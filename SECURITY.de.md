# Sicherheitsrichtlinie

## Meldung von Sicherheitslücken

Bitte melden Sie Sicherheitslücken nicht über öffentliche GitHub-Issues, Diskussionen oder Pull Requests.

Nutzen Sie stattdessen die Funktion zur privaten Meldung von Sicherheitslücken auf GitHub:

1. Öffnen Sie den Reiter **Security** dieses Repositories.
2. Wählen Sie **Report a vulnerability**.
3. Beschreiben Sie die Sicherheitslücke möglichst detailliert.

Hilfreiche Informationen sind insbesondere:

* die betroffene Komponente oder Seite
* Schritte zur Reproduktion des Problems
* die mögliche sicherheitsrelevante Auswirkung
* betroffene Browser, Umgebungen oder Versionen
* bekannte Gegenmaßnahmen oder Workarounds

Bitte greifen Sie nicht auf fremde Daten zu, verändern Sie diese nicht und veröffentlichen Sie keine Daten, die Ihnen nicht gehören.

## Erwartete Reaktionszeiten

Dieses Projekt wird von einem einzelnen Maintainer in seiner Freizeit betreut.

Eingehende Sicherheitsmeldungen werden in der Regel innerhalb von sieben Tagen gesichtet. Eine erste Bewertung kann bis zu vierzehn Tage in Anspruch nehmen. Kritische und offensichtlich ausnutzbare Sicherheitslücken werden nach Möglichkeit bevorzugt behandelt.

Es wird keine feste Frist für die Behebung oder Veröffentlichung eines Updates zugesichert. Der tatsächliche Aufwand hängt von Schweregrad, Komplexität und verfügbarer Zeit ab.

Während der Untersuchung einer Meldung erfolgen möglicherweise nicht regelmäßig Statusmeldungen. Relevante Informationen werden bereitgestellt, sobald wesentliche Fortschritte erzielt wurden.

## Umfang des Sicherheitssupports

Der Sicherheitssupport für dieses Projekt erfolgt anlassbezogen auf Grundlage eingehender Meldungen.

Es wird insbesondere nicht zugesichert:

* eine kontinuierliche Sicherheitsüberwachung
* regelmäßige Penetrationstests
* eine proaktive Suche nach Schwachstellen
* Unterstützung für jede historische Version
* garantierte Fristen für Sicherheitsupdates
* eine langfristige Pflege sämtlicher Funktionen

Meldungen zu rein theoretischen Schwachstellen ohne realistische Sicherheitsauswirkungen können ohne weitere Maßnahmen geschlossen werden.

## Unterstützte Versionen

Sicherheitskorrekturen werden grundsätzlich nur für die aktuelle Version auf Basis des Standard-Branches sowie für die aktuell veröffentlichte Website betrachtet.

Ältere Commits, Forks, modifizierte Deployments oder lokal betriebene Kopien werden nicht unterstützt.

| Version                         | Unterstützt      |
| ------------------------------- | ---------------- |
| Aktuell veröffentlichte Version | Ja               |
| Aktueller Standard-Branch       | Nach Möglichkeit |
| Ältere Versionen oder Commits   | Nein             |

## Verantwortungsvolle Offenlegung

Bitte geben Sie ausreichend Zeit für die Analyse und gegebenenfalls die Behebung einer Sicherheitslücke, bevor Sie diese öffentlich bekannt machen.

Nach der Bewertung einer Meldung können Maintainer und meldende Person gemeinsam einen geeigneten Zeitpunkt für die Veröffentlichung abstimmen. Zur Dokumentation bestätigter Sicherheitslücken können GitHub Security Advisories verwendet werden.

## Sicherheitsupdates

Bestätigte Sicherheitslücken können durch Commits, Pull Requests, Abhängigkeitsaktualisierungen oder GitHub Security Advisories behoben werden.

Nutzerinnen und Nutzer sollten dieses Repository beobachten oder die Commit- und Release-Historie verfolgen, um über sicherheitsrelevante Änderungen informiert zu bleiben.
