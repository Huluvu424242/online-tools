# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.

Instead, use GitHub's private vulnerability reporting feature:

1. Open the **Security** tab of this repository.
2. Select **Report a vulnerability**.
3. Provide as much information as reasonably possible.

Useful information includes:

* the affected component or page
* steps to reproduce the issue
* the potential security impact
* affected browsers, environments, or versions
* possible mitigations, if known

Please avoid accessing, modifying, or publishing data that does not belong to you.

## Response Expectations

This project is maintained by a single volunteer maintainer in limited spare time.

Security reports are normally reviewed within seven days. An initial assessment may take up to fourteen days. Critical and clearly exploitable vulnerabilities will be prioritized when reasonably possible.

No fixed resolution or release deadline is guaranteed. The time required depends on the severity and complexity of the issue and on maintainer availability.

You may not receive frequent status updates while an issue is being investigated. Relevant updates will be provided when meaningful progress has been made.

## Scope of Security Support

Security support for this project is report-driven.

The maintainer does not promise:

* continuous security monitoring
* regular penetration testing
* proactive vulnerability research
* support for every historic version
* guaranteed security patch deadlines
* long-term maintenance of every feature

Reports about theoretical weaknesses without a realistic security impact may be closed without further action.

## Supported Versions

Only the current version available from the default branch and the currently published website are considered for security fixes.

Older commits, forks, modified deployments, and locally hosted copies are not supported.

| Version                   | Supported   |
| ------------------------- | ----------- |
| Current published version | Yes         |
| Current default branch    | Best effort |
| Older versions or commits | No          |

## Coordinated Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing a vulnerability.

After a report has been assessed, the maintainer and reporter may coordinate an appropriate disclosure date. GitHub Security Advisories may be used to prepare and publish information about confirmed vulnerabilities.

## Security Updates

Confirmed vulnerabilities may be addressed through commits, pull requests, dependency updates, or GitHub Security Advisories.

Users should watch this repository or follow its commit and release history to learn about relevant updates.
